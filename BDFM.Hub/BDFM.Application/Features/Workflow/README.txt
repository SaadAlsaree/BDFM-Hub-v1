نعم، هذه نقطة مهمة جدًا وتحدث في الواقع العملي. عندما يتم إحالة مراسلة إلى عدة جهات في نفس الوقت، قد يرغب المُحيل في توجيه هامش (ملاحظة أو توجيه) مختلف لكل جهة من هذه الجهات.

السكيما الحالية تدعم هذا بالفعل، حيث أن حقل InstructionText (الهامش) موجود في جدول WorkflowStep.

كيف يتم ذلك:

عندما تقوم بإحالة مراسلة إلى عدة جهات (مثلاً، الوحدة أ، الوحدة ب، والوحدة ج) في نفس الوقت، فإنك كما ذكرنا سابقًا، ستقوم بإنشاء سجل WorkflowStep منفصل لكل جهة مستلمة.

السجل 1 (إلى الوحدة أ):

ToPrimaryRecipientId: (ID الوحدة أ)

InstructionText: "يرجى اتخاذ الإجراء العاجل وإعلامنا بالنتائج خلال 24 ساعة." (هامش مخصص للوحدة أ)

Status: Pending

... (باقي حقول WorkflowStep)

السجل 2 (إلى الوحدة ب):

ToPrimaryRecipientId: (ID الوحدة ب)

InstructionText: "للعلم والإحاطة، وتضمين الموضوع في التقرير الشهري." (هامش مخصص للوحدة ب)

Status: Pending

... (باقي حقول WorkflowStep)

السجل 3 (إلى الوحدة ج):

ToPrimaryRecipientId: (ID الوحدة ج)

InstructionText: "يرجى مراجعة الجوانب القانونية وتقديم الرأي خلال 3 أيام." (هامش مخصص للوحدة ج)

Status: Pending

... (باقي حقول WorkflowStep)

ماذا عن المستلمين الثانويين (صورة منه إلى)؟

نفس المبدأ ينطبق على المستلمين الثانويين. جدول WorkflowStepSecondaryRecipients الذي لدينا يحتوي أيضًا على حقل InstructionText:

// In YourProject.Domain/Entities/Workflow/WorkflowStepSecondaryRecipient.cs
public class WorkflowStepSecondaryRecipient
{
    // ... (Id, StepId, RecipientType, RecipientId, Purpose) ...
    public string? InstructionText { get; set; } // هامش مخصص للمستلم الثانوي
}


مثال مع مستلمين ثانويين:

لنفترض أنك تحيل كتابًا إلى الدائرة الإدارية (مستلم أساسي) وتريد إرسال نسخة منه إلى:

مدير مكتب رئيس الجهاز (مستلم ثانوي) مع هامش "للعلم والمتابعة الفورية".

قسم التدقيق الداخلي (مستلم ثانوي آخر) مع هامش "يرجى التدقيق في الإجراءات المالية المذكورة".

ستقوم بإنشاء سجل WorkflowStep واحد للمستلم الأساسي (الدائرة الإدارية) مع الهامش الخاص به.

لهذا الـ WorkflowStep، ستقوم بإنشاء سجلين في WorkflowStepSecondaryRecipients:

السجل الثانوي 1:

StepId: (ID الـ WorkflowStep الرئيسي)

RecipientType: User

RecipientId: (ID مدير مكتب رئيس الجهاز)

Purpose: "للعلم والمتابعة"

InstructionText: "للعلم والمتابعة الفورية."

السجل الثانوي 2:

StepId: (ID الـ WorkflowStep الرئيسي)

RecipientType: Unit

RecipientId: (ID قسم التدقيق الداخلي)

Purpose: "للتدقيق"

InstructionText: "يرجى التدقيق في الإجراءات المالية المذكورة."

الخلاصة:

السكيما الحالية تدعم بشكل كامل إمكانية وجود هامش مختلف لكل جهة مستلمة (سواء كانت مستلمًا أساسيًا أو ثانويًا) عند الإحالة.

للمستلمين الأساسيين المتعددين (إحالة متوازية): يتم إنشاء WorkflowStep منفصل لكل مستلم، وكل WorkflowStep يحمل InstructionText الخاص به.

للمستلمين الثانويين المتعددين لنفس الإحالة: يتم إنشاء WorkflowStepSecondaryRecipients منفصل لكل مستلم ثانوي، وكل سجل ثانوي يحمل InstructionText الخاص به.

لذلك، لا تحتاج إلى تعديل السكيما الرئيسية لهذه الميزة. تحتاج فقط إلى التأكد من أن واجهة المستخدم (Frontend) والمنطق في الواجهة الخلفية (Backend) يسمحان للمستخدم المُحيل بتحديد وتخزين هذه الهوامش المخصصة عند اختيار المستلمين.
